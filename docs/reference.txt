DATA FLOW ANALYSIS
===============================================================================

AUTHENTICATION DATA FLOW
------------------------
Component: AuthContext
State Source: localStorage + API
Data Flow:
1. User login → API call → Token storage → Context update
2. Token validation → API call → Context update
3. Logout → Token removal → Context reset → Redirect

Props Passing:
- AuthContext provides: isAuthenticated, user, login, logout, register
- Components consume via useAuth() hook
- ProtectedRoute component checks authentication

State Changes:
- Login: Set token, set user state, update localStorage
- Logout: Clear token, clear user state, clear localStorage
- Token refresh: Validate token on app mount

WISHLIST DATA FLOW
------------------
Component: useFavorites hook
State Source: localStorage + API
Data Flow:
1. Toggle favorite → Update localStorage → API call (future)
2. Load favorites → Get from localStorage → Sync with API (future)

Props Passing:
- HouseCard/ItemCard receive isFavorite, onToggleFavorite
- SavedPage manages selection state

State Changes:
- Add favorite: Update localStorage, update UI state
- Remove favorite: Update localStorage, update UI state

HOUSE LISTINGS DATA FLOW
------------------------
Component: HousesPage
State Source: API service (apiService.ts)
Data Flow:
1. Page load → apiService.houses.getHouses() → Set state
2. Filter change → apiService.houses.getHouses(filters) → Update state
3. Search → apiService.houses.getHouses(search) → Update state

Props Passing:
- HouseCard receives house data, favorite state, toggle function
- Filter components pass filter values up

State Changes:
- Load houses: API call → Set houses state
- Apply filters: Local filtering → Update displayed houses
- Search: Local search → Update displayed houses

MARKETPLACE DATA FLOW
---------------------
Component: MarketplacePage
State Source: API service (apiService.ts)
Data Flow:
1. Page load → apiService.marketplace.getItems() → Set state
2. Category filter → apiService.marketplace.getItems(category) → Update state
3. Search → apiService.marketplace.getItems(search) → Update state

Props Passing:
- ItemCard receives item data, favorite state, toggle function
- Filter components pass filter values up

State Changes:
- Load items: API call → Set items state
- Apply filters: Local filtering → Update displayed items
- Search: Local search → Update displayed items

NOTIFICATION DATA FLOW
----------------------
Component: NotificationContext
State Source: API + Local state
Data Flow:
1. New notification → WebSocket/API → Context update
2. Mark read → API call → Local state update
3. Clear notifications → API call → Context reset

Props Passing:
- Notification components consume via context
- Badge counts from context state

State Changes:
- Add notification: Update context state, show toast
- Mark read: Update context state, API sync
- Clear all: Reset context state, API sync

FORM SUBMISSION DATA FLOW
--------------------------
Component: Various form components
State Source: Local state + API
Data Flow:
1. Form input → Local state update
2. Form submit → Validation → API call → Response handling
3. Success → Toast notification → Navigation/Reset
4. Error → Error display → Keep form data

Props Passing:
- Form components receive submit handlers
- Validation errors passed to UI components

State Changes:
- Input change: Update form state
- Submit: API call, handle response, update UI

HOUSE DATA STRUCTURE
-------------------
interface House {
  id: string                    // Unique identifier
  title: string                 // Property title
  houseType: HouseType          // bedsitter | single | 1br | 2br | 3br
  location: string              // Property location
  price: number                 // Monthly rent in KSh
  deposit: number               // Security deposit in KSh
  description: string           // Property description
  amenities: string[]           // List of amenities
  images: string[]              // Image URLs (1-3 images)
  landlordName: string          // Landlord display name
  landlordId?: string           // Landlord user ID
  phone: string                 // Contact phone
